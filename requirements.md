# Подробные указания по выполнению практической работы.

Решение под **вариант 1** на **Windows 10**: проект **Caddy v2.3.0**,
генерация **SBOM в CycloneDX JSON** и анализ через **Trivy**.

Необходимо выполнить следующие действия (при необходимости):

-   установить **Git**

-   установить **Go**

-   установить **cyclonedx-gomod**

-   установить **Trivy**

-   клонировать **Caddy**

-   переключиться на **v2.3.0**

-   сгенерировать **sbom.json**

-   проверить файл

-   проанализировать его через **Trivy**

## 1. Что нужно установить

Для вашего варианта достаточно 4 вещей:

1.  **Git** --- чтобы скачать репозиторий Caddy.

2.  **Go** --- потому что cyclonedx-gomod работает с Go modules.

3.  **cyclonedx-gomod** --- рекомендованный в задании инструмент для
    Go-проектов; он создает SBOM из Go modules.

4.  **Trivy** --- средство для анализа готового SBOM.

## 2. Установка Git на Windows 10

Откройте **PowerShell** от обычного пользователя и выполните:

winget install \--id Git.Git -e \--source winget

После установки закройте PowerShell и откройте заново.

Проверьте:

git \--version

Если видите версию Git, всё в порядке.

## 3. Установка Go на Windows 10

Скачайте **MSI**-установщик Go с официального сайта Go и установите его
обычным мастером <https://go.dev/dl/>. Официальная инструкция для
Windows говорит: открыть MSI, пройти установку, затем закрыть и заново
открыть командную строку, чтобы переменные среды обновились. По
умолчанию Go ставится в Program Files или Program Files (x86).

После установки откройте новый **PowerShell** и проверьте:

go version

## 4. Установка cyclonedx-gomod

go install
<github.com/CycloneDX/cyclonedx-gomod/cmd/cyclonedx-gomod@latest>

Проверка:

cyclonedx-gomod version

## 5. Установка Trivy

PowerShell

winget install -e \--id AquaSecurity.Trivy

Проверка:

PowerShell

trivy \--version

Если winget не работает, скачай вручную:

-   Перейди на <https://github.com/aquasecurity/trivy/releases/latest>

-   Скачай trivy\_\*.windows-64bit.zip

-   Распакуй trivy.exe в папку, например C:\\trivy

-   Добавь эту папку в PATH (Поиск → «Переменные среды» → Path →
    Добавить).

## 6. Подготовка рабочей папки

Теперь создайте папку для практической работы.

Откройте **PowerShell** и выполните:

mkdir C:\\sbom-lab\
cd C:\\sbom-lab

## 7. Клонирование проекта Caddy

Копируем репозиторий caddyserver/caddy и версия v2.3.0.

Выполните:

git clone https://github.com/caddyserver/caddy.git\
cd caddy\
git checkout v2.3.0

-   git clone --- скачивает репозиторий;

-   cd caddy --- переходит в папку проекта;

-   git checkout v2.3.0 --- переключает проект на нужную версию.

Проверьте, что вы действительно на нужном теге, в папке проекта
выполните:

dir

У вас должен быть файл go.mod, потому что cyclonedx-gomod работает
именно с Go modules. Сам инструмент предназначен для создания SBOM из Go
modules.

## 8. Генерация SBOM

Официально у подкоманды mod есть флаги -json, -output, -licenses, -std.

Выполните в папке C:\\sbom-lab\\caddy:

cyclonedx-gomod mod -json -output sbom.json .

Если хотите чуть богаче SBOM, можно так:

cyclonedx-gomod mod -licenses -std -json -output sbom.json .

Что делают флаги:

-   mod --- строит SBOM для модуля целиком;

-   -json --- формирует результат в JSON;

-   -output sbom.json --- сохраняет результат в файл sbom.json;

-   -licenses --- пытается определить лицензии зависимостей;

-   -std --- добавляет стандартную библиотеку Go как компонент;

-   . --- значит "использовать текущую папку как путь к модулю".

Если команда не находится из PATH, используйте полный путь:

C:\\Tools\\cyclonedx-gomod\\cyclonedx-gomod.exe mod -licenses -std -json
-output sbom.json .

## 9. Проверка, что SBOM действительно создан

После генерации выполните:

dir sbom.json

Потом:

Get-Content .\\sbom.json -TotalCount 30

Что здесь важно увидеть:

-   файл реально существует;

-   внутри JSON;

-   в нем есть поля вроде components, metadata, dependencies.

Это полезно и для самопроверки, и для скриншота в отчет.

## 10. Анализ SBOM через Trivy

Теперь запускаете анализ именно **SBOM-файла**.

Команда:

trivy sbom .\\sbom.json

При первом запуске Trivy, скорее всего, скачает базы.

## 11. Как сохранить результат Trivy в файл

Чтобы потом было удобно вставлять результат в отчет, сразу сохраните
вывод:

trivy sbom .\\sbom.json \> .\\trivy-report.txt

После этого можно открыть файл:

notepad .\\trivy-report.txt

## 12. Идентификация уязвимости и уточнение оценки CVSS

Выбираем из отчета Trivy уязвимость CVE-2021-33194 (golang.org/x/net)

В результате анализа SBOM с использованием Trivy была выявлена
уязвимость в зависимости:

-   **Компонент:** golang.org/x/net

-   **Версия:** v0.0.0-20201110031124-69a78807bb2b

-   **Уязвимость:** **CVE-2021-33194**

-   **Severity:** HIGH

-   **Исправленная версия:** 0.0.0-20210520170846-37e1c6afe023

### CVSS оценка

-   **CVSS v3 Score:** 7.5

-   **Vector:** AV:N/AC:L/PR:N/UI:N/S:U/C:N/I:N/A:H

### Интерпретация:

-   AV:N --- атака удалённая (через сеть)

-   AC:L --- низкая сложность

-   PR:N --- не требуется авторизация

-   A:H --- высокий impact на доступность

Это означает, что уязвимость может использоваться для **DoS-атаки без
аутентификации**

### Описание уязвимости

Уязвимость в golang.org/x/net/html позволяет вызвать бесконечный цикл
при обработке специально сформированного входного HTML (ParseFragment),
что приводит к отказу в обслуживании (DoS).

### Бизнес-риски

-   зависание сервера при обработке входных данных

-   деградация сервиса

-   возможный полный отказ веб-сервера

-   эксплуатация не требует аутентификации

Для веб-сервера (Caddy) это критично, так как он обрабатывает
пользовательские HTTP-запросы.

### Update Plan

Цель: Устранить уязвимость CVE-2021-33194 путём обновления зависимости
golang.org/x/net.

В проекте используется:

golang.org/x/net v0.0.0-20201110031124-69a78807bb2b

Безопасная версия:

0.0.0-20210520170846-37e1c6afe023

Шаг 1 --- обновление зависимости до последней версии

go get <golang.org/x/net@latest>

Шаг 2 --- очистка зависимостей

go mod tidy

Шаг 3 --- фиксация изменений

git add go.mod go.sum

git commit -m \"Fix CVE-2021-33194: update golang.org/x/net\"
